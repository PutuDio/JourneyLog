# ROCTR - JourneyLog Project Overview

## 1. Role (Peran)
* **Product Owner / Fullstack Developer:** Putu Dio & Collaborator
* **Tanggung Jawab:** Merancang database, membangun API backend, mengimplementasikan UI/UX yang mobile-first, serta melakukan deployment ke Vercel.

## 2. Objective (Tujuan)
Membangun Minimum Viable Product (MVP) untuk aplikasi web **JourneyLog** versi 1.0 yang stabil, aman, dan responsif. Aplikasi harus bisa digunakan oleh traveler, daily activity user, dan gym enthusiast untuk mencatat aktivitas mereka dalam satu platform.

## 3. Context (Konteks)
* **Masalah:** Catatan pengguna saat ini tersebar di banyak aplikasi terpisah (notes harian, fitness tracker, travel blog).
* **Solusi:** JourneyLog menyatukannya ke dalam satu aplikasi berbasis kategori (Travel, Daily, Gym) dengan fitur pencarian cepat (< 500ms) dan dukungan multimedia (upload gambar).
* **Tech Stack:** Next.js, TypeScript, Tailwind CSS, shadcn/ui, Prisma, PostgreSQL, Auth.js, Supabase Storage, Vercel.

## 4. Tasks (Daftar Tugas Utama)
* [ ] **Fase 1: Inisialisasi & Setup Lingkungan Kerja** (Repository, Schema Database, Auth)
* [ ] **Fase 2: Pengembangan Backend & API** (Server Actions, Relasi Database, Storage API)
* [ ] **Fase 3: Pengembangan Frontend & UI** (Komponen shadcn/ui, Dashboard, Text Editor)
* [ ] **Fase 4: Integrasi & Pengujian** (Koneksi Frontend-Backend, Uji Responsivitas Mobile)
* [ ] **Fase 5: Deployment & Release** (Konfigurasi Production Vercel & Live Test)

## 5. Result / Resources (Hasil Akhir & Sumber Daya)
* **Hasil Akhir:** Aplikasi web JourneyLog v1.0 live di Vercel dengan seluruh fitur MVP berfungsi 100%.
* **Sumber Daya:** Dokumen PRD v1.0, Repositori GitHub, Akun Supabase, Akun Vercel.
